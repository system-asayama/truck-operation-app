# トラック運行管理アプリ デザイン設計書

## アプリ概要

トラックドライバーが日々の運行を記録・管理するためのモバイルアプリ。外部の Flask バックエンド（samurai-hub.com）と連携し、運行開始・ステータス変更・GPS 追跡・運行履歴確認などの機能を提供する。

## 画面一覧

### 1. 運行画面（ホーム）— `app/(tabs)/index.tsx`

**主要コンテンツ:**
- ネイビーヘッダー（現在日時・リアルタイム時計・ステータスバッジ）
- ドライバー情報カード（名前・スタッフ ID）
- 本日の運行情報カード（トラック名・ルート・開始時刻）
- アクションボタングリッド（運行開始 / 休憩 / 荷積み / 荷下ろし / 運行終了）
- GPS 追跡ステータスカード

**主要機能:**
- 運行開始（トラック選択 → ルート選択 → 開始）
- ステータス変更（運行中 / 休憩中 / 荷積み中 / 荷下ろし中 / 運行完了）
- バックグラウンド GPS 追跡の開始・停止

### 2. 運行履歴画面 — `app/(tabs)/history.tsx`

**主要コンテンツ:**
- ネイビーヘッダー
- 月選択ナビゲーション（前月・翌月ボタン）
- 運行記録カードリスト（日付・ステータス・トラック・ルート・開始/終了時刻・運行時間）

**主要機能:**
- 月別の運行履歴一覧表示
- 運行時間の自動計算

### 3. 設定画面 — `app/(tabs)/settings.tsx`

**主要コンテンツ:**
- ネイビーヘッダー
- ログインフォーム（ログイン ID・パスワード）
- ログイン中ユーザー情報カード（名前・スタッフ ID・サーバー URL・テナント）
- 詳細設定（サーバー URL・テナントスラッグ・モバイル API キー）
- 表示設定（ダークモード切り替え）
- APK ダウンロード（Android のみ）
- アプリ情報

**主要機能:**
- Flask バックエンドへのログイン・ログアウト
- サーバー接続設定の保存（ログアウト後も保持）
- ダークモード切り替え

## 主要ユーザーフロー

### 初回起動フロー
1. 設定タブを開く
2. サーバー URL・テナントスラッグを入力（詳細設定）
3. ログイン ID・パスワードを入力してログイン
4. 運行タブに移動

### 運行開始フロー
1. 運行タブで「運行開始」ボタンをタップ
2. トラック選択モーダルからトラックを選択
3. ルート選択モーダルからルートを選択（または「指定しない」）
4. 運行開始・GPS 追跡が自動的に開始される
5. ステータスボタンで状態を更新

### 運行終了フロー
1. 「運行終了」ボタンをタップ
2. 確認ダイアログで「確認」をタップ
3. GPS 追跡が停止され、運行完了状態になる

## 外部連携

- **Flask バックエンド:** `https://samurai-hub.com`（デフォルト）
- **認証:** `X-Mobile-API-Key` + `X-Staff-Token` ヘッダー
- **GPS 追跡:** `expo-task-manager` によるバックグラウンドタスク
- **オフライン対応:** GPS 送信失敗時は AsyncStorage に最大 200 件バッファリング
